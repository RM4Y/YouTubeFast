YouTube Fast

Fast indexing via the YouTube Data API v3. Playback is resolved on demand with yt-dlp; Jellyfin handles transcoding and hardware acceleration.

Google Cloud Console -> enable "YouTube Data API v3".
A path inside a Jellyfin library, as seen by the server (e.g. /media/youtube).
After each sync, automatically create a "Movies" library named Youtube@{User} pointing at {Library folder}/{User} for every user that has content, if it doesn't already exist.
Address devices use to reach this server (written into .strm files). Use the LAN IP, not localhost.

Quality

Jellyfin then transcodes with whatever HW acceleration you configured in Playback settings.
Skips downloading the full YouTube watch page when resolving a stream (uses the lighter innertube API), so videos start faster. Disable if some videos fail to start.
Leave empty to use the automatic selector. Example: bv*[height<=1080]+ba/b.
Passed to every yt-dlp call. Default: --js-runtimes deno:/usr/bin/deno --ffmpeg-location /usr/lib/jellyfin-ffmpeg/ffmpeg (Deno is required for YouTube's n-challenge so 1080p+ formats resolve). Adjust the deno path to match your container.

Indexing

0 = keep everything. Lower = faster syncs, less quota.
Higher = instant re-plays (no yt-dlp call). YouTube links last ~6h; 300 is a safe default. Keep under 360.
When excluding Shorts, only videos at/below this length are checked with yt-dlp (vertical video = Short). Longer videos are never Shorts. Default 180.
Maximum number of channels shown when users search on the YouTubeFast/app page (1-50). Each search costs 100 YouTube quota units. Default 20.
